ThousandEyes Catalyst 9000 Enablement Guide

A step-by-step workflow for Cisco Portfolio SEs to activate entitlements, deploy agents, and configure monitoring tests.

Each Cisco Catalyst 9300/9400 with a DNA Advantage or Premier subscription includes ThousandEyes Enterprise Agent test credits. This guide walks you through helping a customer go from entitlement to full monitoring visibility.

Entitlement Summary

Each DNA license provides 22 ThousandEyes units per month (enough for 1 network or web test every 5 minutes from one agent). Units pool across all DNA licenses up to a maximum of 110,000 units/month per customer.

⚠
Important
ThousandEyes Cloud Agent access is NOT included in the Cisco DNA license. Cloud agents require a separate ThousandEyes subscription.

Phase 1

Claim ThousandEyes Entitlements

Activate the customer's ThousandEyes entitlements through Cisco Smart Software Manager.

Step 1: Access Cisco Smart Software Manager

1

Navigate to software.cisco.com and log in with the customer's Cisco.com credentials.

2

Click Manage Licenses from the Smart Software Licensing section.

3

Go to Inventory > Licenses in the navigation menu.

Step 2: Locate and Claim Entitlements

1

In the licenses list, find the entry labeled ThousandEyes Enterprise Agent Tests.

2

Select the quantity of DNA licenses to convert for ThousandEyes. The most common approach is to select the full quantity available.

3

Review and submit the upgrade request. This triggers automatic provisioning on the ThousandEyes platform.

Step 3: Confirm the ThousandEyes Account

1

After submission, ThousandEyes provisions an organization for the customer and deposits entitlements.

2

The customer receives a welcome email from ThousandEyes at the email address on their Smart Account.

3

The customer clicks the confirmation link to set up credentials and access the dashboard.

💡
SE Tip
Ensure the customer checks their inbox (and spam folder) for the email associated with their CSSM account. If they have an existing ThousandEyes org, the account admin receives a subscription update notification instead.
Phase 2

Share Access with Your SE (Optional)

Grant the SE dashboard access for collaborative troubleshooting and configuration.

Adding Users to the ThousandEyes Organization

1

The customer's ThousandEyes admin navigates to Account Settings > Users and Roles > Users.

2

Click + New Users.

3

Enter the SE's email address. Select an Account Group and assign a role.

4

Click Add New Users. The SE receives an email invitation.

Role Comparison

RoleUse CasePermissions
Regular UserView-only for the SERead-only: tests, results, dashboards
Account AdminSE helps configure testsManage tests, agents, alerts within account group
Organization AdminFull collaboration (rare)Full admin across all account groups
💡
SE Tip
For most engagements, request Account Admin on a specific account group. This gives you enough access to configure and troubleshoot without full org control. The customer retains ownership.
Phase 3

Deploy ThousandEyes Enterprise Agents

Install agents on Catalyst 9000 switches via CLI or Catalyst Center.

Pre-Deployment Checklist

  • IOS-XE version meets minimum (17.3.3 for Cat 9300, 17.5.1 for Cat 9400)
  • VLAN identified for Enterprise Agent traffic (not default VLAN)
  • IP addressing planned (static recommended for production)
  • Outbound TCP 443, DNS, and NTP confirmed from agent VLAN
  • Account group token obtained from ThousandEyes: Cloud & Enterprise Agents > Agent Settings > Add New Enterprise Agent > Cisco Application Hosting

Ideal for individual switch deployments or when Catalyst Center is not in use.

Step 1: Enable IOx Application Framework

configure terminal
iox
end

Verify IOx is running:

show iox-service

Step 2: Download and Install the Agent Image

If the image is not preloaded, copy to flash and install:

app-hosting install appid thousandeyes_agent package flash:thousandeyes-enterprise-agent-<version>.cisco.tar

Or install directly via HTTPS (if switch has internet access):

app-hosting install appid thousandeyes_agent package https://downloads.thousandeyes.com/enterprise-agent/thousandeyes-enterprise-agent-<version>.cisco.tar

Step 3: Configure the VLAN and AppGigabitEthernet Interface

configure terminal
!
vlan 100
 name TE-Agent-VLAN
!
interface GigabitEthernet1/0/1
 description Uplink for TE Agent
 switchport access vlan 100
!
interface AppGigabitEthernet1/0/1
 switchport trunk allowed vlan 100
 switchport mode trunk
end

Step 4: Configure Application Hosting Parameters

Replace placeholder values with the customer's environment details:

configure terminal
app-hosting appid thousandeyes_agent
 app-vnic AppGigabitEthernet trunk
  vlan 100 guest-interface 0
   guest-ipaddress 10.10.100.50 netmask 255.255.255.0
  exit
 exit
 app-default-gateway 10.10.100.1 guest-interface 0
 name-server0 10.10.100.10
 name-server1 8.8.8.8
 app-resource docker
  prepend-pkg-opts
  run-opts 1 "-e TEAGENT_ACCOUNT_TOKEN=<your-token-here>"
  run-opts 2 "--hostname Cat9K-Branch-Agent"
 exit
 start
end
write memory

Step 5: Verify the Agent

show app-hosting list

The output should show the agent in RUNNING state. For detailed information:

show app-hosting detail appid thousandeyes_agent

The agent should appear in the ThousandEyes portal under Cloud & Enterprise Agents > Agent Settings within a few minutes.

🔎
Troubleshooting
If the agent doesn't appear, connect to the container shell: app-hosting connect appid thousandeyes_agent session, then check /var/log/agent/te-agent.log. Common issues: incorrect token, missing DNS resolution, or blocked outbound TCP 443.

Catalyst Center provides a streamlined GUI workflow for deploying agents across multiple switches simultaneously.

Step 1: Integrate ThousandEyes with Catalyst Center

1

Navigate to System > Settings > External Services > ThousandEyes Integration.

2

Paste the ThousandEyes Account Group Token from the TE portal.

3

Click Save. The page confirms integration is enabled.

Step 2: Upload the Agent Image

1

Download the Cisco IOS XE Docker Application (.tar) from the ThousandEyes portal > Catalyst Switches tab.

2

In Catalyst Center, navigate to Provision > App Hosting for Switches.

3

Upload the downloaded .tar file.

Step 3: Configure Switch Prerequisites

1

Select target switch(es) from inventory.

2

Configure the VLAN assignment (Catalyst Center shows available VLANs).

3

Ensure AppGigabitEthernet interfaces are trunked on target switches.

4

Enable HTTPS on the switch with local authentication for Catalyst Center verification.

Step 4: Deploy the Agent

1

From the App Hosting workflow, assign the ThousandEyes app to target switches.

2

Configure IP settings, Docker run options (token, hostname, proxy if needed).

3

Click Deploy. Catalyst Center pushes the config to all selected switches simultaneously.

Step 5: Verify in ThousandEyes Portal

After deployment, verify agents appear under Cloud & Enterprise Agents > Agent Settings. Agents should register within a few minutes.

💡
SE Tip
Ensure the switch hostname in Catalyst Center inventory matches exactly what's configured on the switch. FQDN mismatches are the most common integration issue.
Phase 4

Create Monitoring Tests

Configure network, application, and DNS monitoring tests for immediate visibility.

In the ThousandEyes portal, navigate to Network & App Synthetics > Test Settings and click Start Monitoring (or use the Guided Setup wizard).

Test 1: Agent-to-Server (Network Path Monitoring)

Use Case

Monitor the network path from a remote branch (Catalyst 9000 agent) to a data center gateway, WAN concentrator, or cloud endpoint. Provides visibility into packet loss, latency spikes, and routing changes.

Configuration

SettingValueNotes
Test TypeAgent to ServerUnder the Network category
Test Name(descriptive)e.g., "Branch-NYC to DC-Dallas Network"
TargetServer IP or hostnamee.g., data center gateway IP or hostname:port
ProtocolTCP (recommended)Options: TCP, ICMP, Auto-detect
Interval5 minutesConsumes 1 unit per interval per agent
AgentsSelect Cat 9K agent(s)Choose from Enterprise Agents list
AlertsEnable default rulesCustomize thresholds as needed

What You Get

  • End-to-end metrics: packet loss, latency, jitter, and path MTU
  • Path Visualization: traceroute-style map with every hop, IPs, MPLS labels, and ASN info
  • Alerting on threshold breaches for proactive monitoring

Test 2: HTTP Server (Application Monitoring)

Use Case

Monitor availability and performance of critical web applications (internal ERP, Salesforce, M365 login, etc.) from the branch network perspective.

Configuration

SettingValueNotes
Test TypeHTTP ServerUnder the Web category
Test Name(descriptive)e.g., "Branch-NYC to Salesforce HTTP"
URLFull target URLe.g., https://login.salesforce.com
HTTP MethodGET (default)Use POST for API endpoints

Network MeasurementsEnabledAdds agent-to-server network metrics
SSL VerificationEnabled (default)Disable only for self-signed certs
AlertsEnable default rulesCustomize thresholds as needed

What You Get

  • Availability percentage and HTTP response codes
  • Timing breakdown: DNS, TCP connect, SSL/TLS, TTFB, total response time
  • Network metrics (when enabled): latency, loss, jitter, path visualization
  • Content verification (optional): validate expected response content

Test 3: DNS Server (DNS Monitoring)

Use Case

Monitor internal DNS servers or public DNS to ensure critical domains resolve correctly from branch locations. Valuable for split-horizon DNS or internal-only domains.

  • DNS availability percentage per server
  • Resolution time for each queried server
  • Resolution results (returned IPs) for validation
  • Network metrics to each DNS server (when enabled)
💡
SE Tip: Starter Package
Recommend one of each test type as a starter: an agent-to-server test to the data center for WAN visibility, an HTTP server test to their most critical web app, and a DNS test to primary DNS infrastructure. This provides a well-rounded baseline that demonstrates immediate value.
Phase 5

Validate and Next Steps

Confirm everything is working and plan for expanded monitoring.

Validation Checklist

  • Agent appears in ThousandEyes portal under Cloud & Enterprise Agents > Agent Settings
  • Agent status shows "Online" with green indicator
  • All configured tests show data within 5-10 minutes of creation
  • Path Visualization displays hop-by-hop data for network and HTTP tests
  • DNS test results show successful resolution with expected IPs
  • Alert rules are active and configured

Recommended Next Steps

  • Walk the customer through dashboard views (Overview, Path Visualization, Table)
  • Set up a custom dashboard for at-a-glance monitoring
  • Configure alert notifications (email, Slack, PagerDuty, webhooks)
  • Add tests as the customer identifies more critical paths and applications
  • Explore the ThousandEyes / Catalyst Center integration for unified visibility
  • Deploy agents on additional Catalyst switches at other branch sites
